❓ Gözlənilən Suallar və Cavablar

Bu platformanı mövcud olanlardan (Discord, Slack) nə fərqləndirir?
Discord və Slack kommunikasiya alətləridir — mesajlaşma üçündür. Biz isə engagement management platformasıyıq. Heç bir platforma üzvlərin nə vaxt "churned" olacağını əvvəlcədən proqnozlamır, psixoloji prinsiplərə əsaslanan re-engagement bildirişləri göndərmir, və ya streak/gamification ilə geri qaytarmır. Biz Discord-un əvəzi deyilik — Discord-un üzərinə əlavə olunan analitika və retention təbəqəsiyik.
Necə pul qazanacaqsınız? Biznes modeli nədir?
3 gəlir mənbəyi:
1) Freemium abunəlik: Free plan 2 icma + 50 üzv limiti. Pro ($19/ay) — AI analitika, unlimited üzv. Enterprise ($79/ay) — API access, white-label, SSO.
2) Hədəflənmiş reklamlar: Şirkətlər üzvlərin bacarıqlarına uyğun iş elanları yerləşdirir → CPC (click başına ödəniş) model.
3) API inteqrasiyası: Enterprise müştərilər bot inteqrasiyası üçün API access alır.
İstifadəçinin aktiv olduğunu haradan bilirsiniz?
Check-in sistemi ilə. Hər gün istifadəçi platformaya daxil olub check-in edir → streak artır. Check-in etməsə streak sıfırlanır. Bu Duolingo modelidir — sübut olunmuş retention mexanizmasıdır. Əlavə olaraq, tədbirlərə qeydiyyat və iştirak, icmaya qoşulma, badge qazanma kimi hər fəaliyyət izlənir və xal verilir. 42 gün check-in etməyən istifadəçi avtomatik olaraq "CHURNED" statusu alır.
AI cavabları real API-dən gəlir, yoxsa mock datadır?
Real Google Gemini 2.5 Flash API-dən gəlir. Backend-də GeminiAiService var — 3 funksiyası var: community health recommendations, psychological notification generation, və individual member analysis. Əgər API əlçatan deyilsə (rate limit və s.), avtomatik fallback mexanizmi var — əvvəlcədən hazırlanmış keyfiyyətli cavablar göstərilir. Yəni istifadəçi heç vaxt boş ekran görmür.
Niyə H2 database istifadə edirsiniz? Produksiyada nə olacaq?
H2 yalnız development və demo üçündür — sürətli, sıfır konfiqurasiya, in-memory. Produksiyada PostgreSQL-ə keçid 1 sətirlik konfiqurasiya dəyişikliyidir. JPA/Hibernate istifadə etdiyimiz üçün heç bir kod dəyişikliyi lazım deyil — yalnız application.properties-dəki datasource URL dəyişir.
Notification necə göndərilir? Push notification var?
Hazırda bildirişlər in-app notification olaraq database-ə yazılır və istifadəçi Notifications səhifəsində görür. AI Insights-dan yaradılan bildirişlər birbaşa həmin istifadəçinin notification siyahısına düşür. Gələcək planlarımızda: Firebase Cloud Messaging ilə push notification, email notification inteqrasiyası, və Telegram/Discord webhook-ları var.
Psixoloji bildirişlər dediyiniz nədir? Etik problemlər yoxdur?
Nir Eyal-ın "Hooked" kitabından ilhamlanmışıq — Trigger → Action → Variable Reward → Investment döngüsü. Məsələn, "42 gün görmədik, dostların aktivdir" (FOMO), "8 günlük streak-in əfsanəvi idi" (nostalgia + itirilmə). Etik tərəf: biz manipulyasiya deyil, motivasiya edirik. Duolingo, Spotify Wrapped, GitHub contribution chart — hamısı eyni prinsiplərdən istifadə edir. Bizim məqsəd istifadəçini zorla qaytarmaq deyil, dəyəri xatırlatmaqdır.
İstifadəçi digər platformalardan (Google, GitHub) login olsa, onları necə izləyəcəksiniz?
Hazırda JWT-based öz autentifikasiya sistemimiz var. OAuth2 inteqrasiyası (Google, GitHub login) Spring Security OAuth2 Client ilə asanlıqla əlavə olunur — roadmap-da var. İstifadəçi Google ilə login olsa belə, platformadakı bütün fəaliyyəti — check-in, event attendance, xal — bizim database-də izlənir. Xarici platformadan yalnız autentifikasiya alırıq, engagement tracking tamamilə bizimdir.
Security necə təmin olunub?
Spring Security + JWT token autentifikasiyası. Hər API sorğusu Bearer token tələb edir. Şifrələr BCrypt ilə hash olunur. CORS konfiqurasiyası var. Owner-only əməliyyatlar (community delete, member kick, event delete) service səviyyəsində yoxlanır — yalnız owner və ya admin icazə alır. SQL injection-dan qorunma JPA/Hibernate tərəfindən avtomatik təmin olunur.
Layihənin scalability-si necədir? 10.000 istifadəçiyə dözə bilər?
Arxitektura stateless-dir — JWT token istifadə etdiyimiz üçün horizontal scaling mümkündür. PostgreSQL + connection pooling (HikariCP default gəlir) minlərlə concurrent istifadəçini dəstəkləyir. Gemini API sorğuları async WebClient ilə edilir — blocking yoxdur. Gələcəkdə Redis cache əlavə etməklə 100K+ istifadəçini rahat dəstəkləyə bilərik.
Bu layihəni real dünyada tətbiq etmək üçün nə lazımdır?
3 addım:
1) PostgreSQL database + production server (AWS/DigitalOcean — $20/ay)
2) Gemini API key (Google AI Studio-dan pulsuz tier var)
3) Domain + SSL sertifikatı
Cəmi 2-3 həftə əlavə iş ilə produksiyaya çıxarıla bilər. Kod artıq produksiya keyfiyyətindədir — error handling, fallback, role-based access control hamısı var.
